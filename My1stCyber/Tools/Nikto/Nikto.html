About Nikto

Nikto is an open-source web server scanner that performs comprehensive tests against web servers for multiple items, including dangerous files/CGIs, outdated versions of servers, and version-specific problems.

Key Features

  • Scans for over 6700 potentially dangerous files/CGIs
  • Checks for outdated server versions
  • Identifies installed server modules and software
  • SSL support for testing HTTPS servers
  • Multiple report formats (text, HTML, XML, etc.)
  • Proxy support for scanning through firewalls
  • Customizable through plugins and configuration

Installation

On Kali Linux

sudo apt update && sudo apt install nikto

Using Git

git clone https://github.com/sullo/nikto.git

Verify Installation

nikto -Version

Basic Usage Guide

1. Basic Scan

Perform a basic scan of a website:

nikto -h http://example.com

2. HTTPS Scan

Scan an HTTPS website:

nikto -h https://example.com -ssl

3. Save Output

Save scan results to a file:

nikto -h http://example.com -o report.html -Format html

Advanced Usage

1. Port Scanning

Scan specific port:

nikto -h example.com -p 8080

Scan multiple ports:

nikto -h example.com -p 80,443,8080

2. Authentication

Basic authentication:

nikto -h example.com -id admin:password

3. Proxy Usage

Use a proxy:

nikto -h example.com -useproxy http://proxy:8080

4. Tuning Options

  • Test specific vulnerabilities: nikto -h example.com -Tuning 1,2,3
  • Exclude certain tests: nikto -h example.com -Tuning x

Tuning Options Reference

  • 0 - File Upload
  • 1 - Interesting File / Seen in logs
  • 2 - Misconfiguration / Default File
  • 3 - Information Disclosure
  • 4 - Injection (XSS/Script/HTML)
  • 5 - Remote File Retrieval
  • 6 - Denial of Service
  • 7 - Remote File Retrieval
  • 8 - Command Execution / Remote Shell
  • 9 - SQL Injection
  • a - Authentication Bypass
  • b - Software Identification
  • c - Remote Source Inclusion
  • x - Reverse Tuning Options

Practical Examples

Comprehensive Scan

nikto -h example.com -C all

Stealthy Scan

nikto -h example.com -evasion 1

Full Report with Multiple Formats

nikto -h example.com -o report.html -Format htm,csv,txt

Best Practices

  • Legal Compliance: Only scan systems you have permission to test
  • Documentation: Always save scan results for future reference
  • Performance: Use tuning options to focus scans on relevant tests
  • Validation: Manually verify findings to eliminate false positives
  • Updates: Keep Nikto and its databases updated

Common Nikto Flags

  • -h: Target host
  • -p: Port number(s)
  • -ssl: Force SSL mode
  • -nossl: Disable SSL mode
  • -output: Save output to file
  • -Format: Output format
  • -Tuning: Scan tuning
  • -timeout: Response timeout
  • -evasion: Evasion technique
  • -update: Update databases

Troubleshooting

  • Timeouts: Adjust timeout settings with -timeout option
  • False Positives: Use tuning options to reduce false positives
  • Blocked Requests: Try different evasion techniques
  • SSL Errors: Verify SSL configuration and certificates
